# C500 BF16 Perf Probe Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a standalone C500-only BF16 performance probe that measures device-event throughput for pure MMA, MMA plus shared-memory operand feeding, MMA plus global-to-shared staging, and the current full TK GEMM path.

**Architecture:** Keep the benchmark outside the current GEMM implementation and reuse the existing C500/TK internal primitives already validated in the tree: native MMA atoms, `bf16_stage_ring`, aligned raw-stage bridge consumption, and `run_bf16_mainloop`. The probe should report comparable BF16 TFLOP/s numbers without introducing fallback kernels or external GEMM APIs.

**Tech Stack:** C++20, ThunderKittens C500 backend, MXMACA `cucc`, `cudaEvent` device timing.
