# ThunderKittens RTX 4080 Tests Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a runnable `tests/` path for the local RTX 4080 machine and get the public unit-test suite building and running on this host.

**Architecture:** Keep the library behavior split as it already is today: Ampere-like path versus Hopper/Blackwell-only path. Add an explicit `RTX4080` build target that compiles the test suite for `sm_89`, and make multi-GPU test coverage opt-in through `NUM_GPUS` instead of forcing it on by default.

**Tech Stack:** GNU Make, NVCC 12.6, CUDA C++20, ThunderKittens unit tests

---

### Task 1: Add a local 4080 build target for `tests/`

**Files:**
- Create: `docs/superpowers/plans/2026-04-02-rtx4080-tests.md`
- Modify: `tests/Makefile`

- [ ] **Step 1: Reproduce the missing/incorrect 4080 build behavior**

Run: `make GPU_TARGET=RTX4080 -C tests -j4 unit_tests`
Expected: compile starts without a matching architecture selection and fails before producing `tests/unit_tests`

- [ ] **Step 2: Reproduce the existing Ampere target failure**

Run: `make GPU_TARGET=A100 -C tests -j4 unit_tests`
Expected: FAIL with `Unsupported gpu architecture 'compute_80a'`

- [ ] **Step 3: Add an explicit 4080 target and clean up Ampere flags**

Update `tests/Makefile` so that:

```make
GPU_TARGET ?= RTX4080
NUM_GPUS ?= 1
NVCCFLAGS += -DTEST_INTENSITY=2 -DTEST_ALL -DNUM_GPUS=$(NUM_GPUS)
ifneq ($(NUM_GPUS),1)
NVCCFLAGS += -DTEST_ALL_MULTI_GPU
endif

ifeq ($(GPU_TARGET),A100)
NVCCFLAGS += -gencode arch=compute_80,code=sm_80 -DKITTENS_AMPERE -lcuda -lcudart
else ifeq ($(GPU_TARGET),RTX4080)
NVCCFLAGS += -gencode arch=compute_89,code=sm_89 -DKITTENS_AMPERE -lcuda -lcudart
```

- [ ] **Step 4: Rebuild to verify the new target is recognized**

Run: `make GPU_TARGET=RTX4080 NUM_GPUS=1 -C tests -j4 unit_tests`
Expected: compilation proceeds past the old architecture-selection failure

### Task 2: Make multi-GPU tests configurable instead of forced-on

**Files:**
- Modify: `tests/testing_commons/testing_flags.cuh`

- [ ] **Step 1: Add a failing config-driven expectation**

Run: `make GPU_TARGET=RTX4080 NUM_GPUS=1 -C tests -j4 unit_tests`
Expected: if the build still compiles multi-GPU code unconditionally, failures should reference `TEST_ALL_MULTI_GPU`, `NUM_GPUS`, or multi-device paths

- [ ] **Step 2: Make `NUM_GPUS` overridable from the build**

Update `tests/testing_commons/testing_flags.cuh` so that:

```c++
#ifndef NUM_GPUS
#define NUM_GPUS 2
#endif
```

This preserves the old compile-time default for callers that do not define `NUM_GPUS`, while letting `tests/Makefile` set `NUM_GPUS=1` on the local single-GPU machine.

- [ ] **Step 3: Rebuild the single-GPU suite**

Run: `make GPU_TARGET=RTX4080 NUM_GPUS=1 -C tests -j4 unit_tests`
Expected: build no longer enters the forced multi-GPU path by default
